# CODEX_LOG

## 開発の流れ

- 開発基盤として `docs/ai/` を作成。
- HTML / CSS / JavaScript / Canvas の最小ゲームを作成。
- 図形描画から画像素材描画へ移行。
- 画像余白による判定ズレを確認し、hitboxを手動調整。
- `debugMode` でhitbox表示を切り替えられるようにした。
- Canvasを `400 x 600` から `800 x 450` へ横長化。
- 横長化後、障害物速度と初回障害物位置を調整。
- スコアに応じた障害物速度上昇を追加。
- 強ジャンプ機能は試したが、最終的に削除して通常ジャンプのみへ戻した。
- 引き継ぎ用に `HANDOFF.md` を作成し、今回 `STATUS.md` / `CODEX_LOG.md` も追加した。
- 今後ほかのゲームを同じプロジェクト配下に追加できるよう、現在のゲーム本体を `flappy-bird/` にまとめた。

## 重要な判断と理由

- 画像素材はCodex側で生成・編集した。
  - 既存素材は余白が大きく、当たり判定と見た目がズレたため。
- 当たり判定はピクセル判定ではなく四角判定のままにした。
  - 最小プロトタイプとして分かりやすく、調整しやすいため。
- hitbox値は定数化した。
  - 見た目に合わせて後から調整する前提のため。
- デバッグ枠は削除せず `debugMode` で非表示にした。
  - 今後も判定確認に使えるため。
- 強ジャンプは削除した。
  - 操作を単純にし、通常ジャンプだけで調整しやすくするため。
- 難易度上昇はまず速度だけにした。
  - `gapHeight` まで同時に変えると原因が分かりにくいため。

## 現在の問題

- スコア50まで行っても速度上昇をあまり感じない。
- 速度上限 `maxObstacleSpeed = 6` では横長画面だとまだ控えめに感じる可能性がある。
- `gapHeight = 160` は固定で、後半難易度に影響していない。
- 背景画像は `800 x 450` 専用ではない。

## 今後の改善候補

- 速度上昇を一定スコアごとの段階制にする。
- 速度上限を再検討する。
- `gapHeight` をスコアに応じて少しずつ狭くする。
- UIを見やすくする。
- ジャンプ、スコア、ゲームオーバーにサウンドを追加する。
- 背景画像を横長に最適化する。

## 開発で得た知見

- 画像素材は表示サイズに近い寸法で作ると扱いやすい。
- 透明余白が大きい画像は、四角判定と見た目がズレやすい。
- まずhitboxを表示して、見た目と判定のズレを確認すると安全。
- 操作感や難易度は、定数を少しずつ変えて確認するのがよい。
- 画像、hitbox、操作感、難易度は同時に変更しない方が原因を追いやすい。
- 横長化すると、同じ速度でもテンポが遅く感じやすい。
